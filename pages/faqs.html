FAQs

Frequently Asked Questions.

1) Does CBT work with different types of projects as is or does it require separate modules?

Depending on what project type you want to build, you'll need to include separate modules. In most cases, you should be able to leverage existing packages either from an existing build toolset or packages on NuGet.org. For example, Visual Studio SDK projects can already use packages available on NuGet. We highly recommend that you do the same as this will benefit you and future teams!

2) Where do I specify the Nuget package sources?

For project-level assembly references, you can place these in the project directory in a packages.config or project.json. This allows you to manage packages using the Visual Studio Nuget Package Manager. If your project tree is too big and managing packages at the project-level becomes overwhelming, then you can add a $(CBTLocalPath)\GlobalPackages\packages.config which can contain packages for the whole tree. Packages are restored and properties are generated by the CBT.NuGet module so you can add references in your project with paths $(NuGetPath_Package_Id)\lib\net45\assembly.dll
